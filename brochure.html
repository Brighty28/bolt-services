How the CMS Works

All site content lives in a single JSON file. The admin panel provides a user-friendly interface to edit this file. When you click save, the server updates the file and the website reflects the changes immediately -no rebuild or restart needed.

Working Contact Form

The contact form now sends real emails via your existing SMTP service.

Email Delivery

Enquiries are sent directly to your inbox. Each email includes the sender's name, email, subject, and message with a reply-to header for easy responses.

Secure Credentials

SMTP credentials are stored as server environment variables, never in the code. Safe and easy to update without touching the codebase.

Key Benefits

  • Zero-downtime deployments - push to GitHub and the site updates automatically
  • No database dependency - content is file-based, fast, and simple to back up
  • No third-party CMS subscriptions - everything is self-hosted
  • Secure by default - admin panel locked behind authentication in production
  • Lightweight & fast -no frontend frameworks, minimal dependencies

Next Steps

The following items are needed to take the site live.

  • Confirm Node.js extension is available on your Plesk hosting plan
  • Set the deploy branch in Plesk and configure the GitHub webhook
  • Set CMS admin username and password as environment variables
  • Set SMTP credentials as environment variables for the contact form
  • Add your Facebook and LinkedIn page URLs via the CMS Social tab
  • Review all site content in the CMS and update as needed
  • Upload the final hero background image via the Media Library
  • Test the contact form end-to-end in production
  • Point your domain DNS to the hosting server
  • Verify the site on mobile, tablet, and desktop

Environment Variables Required

These must be set in Plesk's Node.js settings before going live:

  • CMS_ADMIN_USER - Username for the admin panel
  • CMS_ADMIN_PASS - Password for the admin panel
  • SMTP_HOST -Email server (send.one.com)
  • SMTP_PORT -Email port (587)
  • SMTP_USER -Email account username
  • SMTP_PASS -Email account password
  • SMTP_FROM -Sender address for contact form emails
  • SMTP_TO -Where contact form enquiries are delivered

Ready to Go Live

Once hosting is confirmed and environment variables are set, the site can be deployed immediately.
